Comenzando

¡Bienvenido a Processing para Android! En este tutorial, aprenderás cómo usar el modo Android en Processing para ejecutar sus bosquejos en un dispositivo Android o en el emulador.

Instalando el SDK

La primera vez que abra el modo Android después de instalarlo a través del CM o manualmente, verá esta ventana, que le pedirá que busque un SDK de Android preinstalado o descargue el SDK de Android automáticamente:

Si aún desea utilizar un SDK ya descargado en su computadora, elija la opción "Localizar la ruta del SDK manualmente" y seleccione la carpeta que contiene el SDK. Sin embargo, este SDK debe incluir Android 8.0 (nivel de API 26) o superior. Para descargar e instalar manualmente el SDK, vaya a esta página, desplácese hacia abajo a la sección de herramientas de la línea de comandos y seleccione la que corresponde a su sistema operativo. Hacer esto podría ser conveniente si planea usar el mismo SDK de diferentes herramientas de desarrollo (línea de comandos, Android Studio, etc.) además de Procesamiento. Sin embargo, tenga en cuenta que el SDK que se descarga e instala el Procesamiento se almacena automáticamente en la carpeta del modo y no interfiere con ningún otro SDK instalado en su computadora.

Si el SDK se localizó o instaló correctamente, debería ver su dispositivo Android en la lista "Seleccionar dispositivo" después de conectarlo a la computadora:

Escribiendo un simple bosquejo

Escribamos un breve bosquejo de procesamiento para que se ejecute en su teléfono o tableta Android y nos aseguremos de que todo funcione como se espera. En el siguiente código, pintamos la mitad de la pantalla que recibe una pulsación con el color negro. Tenga en cuenta que nada en este bosquejo es realmente específico de Android, por lo que también podría ejecutarlo en el modo Java. Esta portabilidad de los bosquejos de procesamiento (siempre que no dependan de ninguna característica específica de la plataforma) es útil para las pruebas, ya que el modo Java incluye un depurador integrado que puede ayudar a detectar errores en el código con mayor facilidad. 


void setup() {
  fullScreen();
  noStroke();
  fill(0);
}

void draw() {
  background(204);
  if (mousePressed) {
    if (mouseX < width/2) {
      rect(0, 0, width/2, height); // Left
    } else {
      rect(width/2, 0, width/2, height); // Right
    }
  }
}

Ejecutando el bosquejo en tu dispositivo

Antes de presionar el botón Ejecutar o seleccionar "Ejecutar en dispositivo" en el menú bosquejo, asegúrese de:

  • Active "Depuración de USB" en el dispositivo: este proceso varía según el dispositivo y la versión del sistema operativo que haya instalado. En la mayoría de los casos, esta configuración se encuentra en Opciones de desarrollador, en Configuración del sistema. En Android 4.2 y versiones posteriores, las Opciones del desarrollador están ocultas de manera predeterminada, pero puede habilitarlas siguiendo estas instrucciones.
  • Si está ejecutando Windows, debe seguir documentación de Google para instalar un controlador USB especial y cuidar configuraciones adicionales. Si descargó el SDK de Android automáticamente en Procesamiento, los últimos controladores USB estarán dentro de la carpeta de modo, debajo de la subcarpeta temporal.

Ejecutando el croquis en el emulador

Si no tiene un dispositivo para ejecutar su bosquejo, aún puede usar el emulador de software. La primera vez que ejecute un croquis en el emulador, Processing le solicitará que descargue la imagen del sistema que necesita el emulador para funcionar:

Después de finalizar la descarga, que puede demorar varios minutos dependiendo de su conexión a Internet (las imágenes del sistema tienen un tamaño de aproximadamente 1 GB), el procesamiento iniciará el emulador y luego iniciará el bosquejo.

¡Otro consejo importante es no cerrar el emulador después de probar un bosquejo, simplemente déjalo abierto para que no tengas que esperar a que el emulador se reinicie la próxima vez!
